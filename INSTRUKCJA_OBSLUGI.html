Przewodnik po zarządzaniu stroną

Niniejsza instrukcja pomoże Ci w samodzielnej aktualizacji treści, zarządzaniu galerią oraz publikacji zmian na serwerze.

Galeria zdjęć

Strona posiada inteligentny mechanizm zarządzania obrazami. Nie musisz dopisywać żadnego kodu, aby nowe zdjęcia pojawiły się w galerii – wystarczy, że umieścisz pliki w odpowiednim folderze.

folder Folder docelowy
terminal src/assets/images/galeria/
  • Format .webp (zalecany)
  • Max. 500 KB na zdjęcie
  • Dowolna nazwa pliku
  • Automatyczne skanowanie

Godziny otwarcia

Informacje o godzinach otwarcia są przechowywane w dwóch miejscach: jedno służy do wyświetlenia tekstu na podstronie Kontakt, a drugie informuje wyszukiwarkę Google (Schema SEO).

1

Aktualizacja na stronie

Otwórz plik src/pages/kontakt.astro i odszukaj sekcję Godziny otwarcia (linia ok. 42). Zmień tekst widoczny dla pacjentów.

2

Aktualizacja dla Google (SEO)

To bardzo ważne dla pozycjonowania. Otwórz src/layouts/BaseLayout.astro i zaktualizuj wartości w sekcji openingHoursSpecification.

info

Dlaczego dwa miejsca?

Google wymaga specjalnego formatu danych (JSON-LD), aby wyświetlać godziny bezpośrednio w wynikach wyszukiwania, podczas gdy strona Kontakt wymaga formatu czytelnego dla człowieka.

Dane kontaktowe

Wszystkie dane teleadresowe są zgrupowane w jednym pliku, co ułatwia ich szybką edycję.

  • Email: Szukaj adresu w linii ok. 32

    • Publikacja strony

    Strona korzysta z generatora Astro. Aby zmiany były widoczne w sieci, należy wygenerować gotowe pliki i wgrać je na serwer.

    1

    Podgląd zmian

    Uruchom lokalny serwer, aby sprawdzić jak wyglądają zmiany przed publikacją:

    npm run dev
    2

    Budowanie projektu

    Wygeneruj gotowe pliki do wysłania na serwer:

    npm run build
    warning

    Gdzie są pliki do wysłania?

    Po wykonaniu komendy npm run build, w głównym folderze powstanie folder dist/. To jego zawartość (a nie cały projekt) należy skopiować na serwer FTP.

    Struktura plików

    Najważniejsze foldery, o których warto wiedzieć:

    src/pages/

    Główne podstrony serwisu. Edytuj je, aby zmienić treści na stronie.

    src/components/

    Elementy powtarzalne, takie jak Menu, Stopka czy przyciski.

    src/assets/

    Zdjęcia, ikony oraz style CSS używane w projekcie.

    public/

    Pliki statyczne, które nie są przetwarzane (np. favicon, ikony aplikacji).
